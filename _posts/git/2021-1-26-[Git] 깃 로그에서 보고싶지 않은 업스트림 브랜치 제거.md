로그에서 보고 싶은 업스트림 브랜치만 보는 방법에 대한 글입니다.

Sourcetree는 깃 로그 그래프가 보기 좋게 만들어서 보여주기 때문에 Sourcetree를 사용하여 Git을 관리하는 경우가 많습니다. 
하지만 Sourcetree의 어떤 설정 때문에 Upstream을 보면 Upstream이 모든 브랜치를 가지고 있습니다. 
터미널로 로그를 볼 때까지만 해도 Upstream 브랜치가 원하는 브랜치만을 가져왔지만 Sourcetree로 보는 순간 모든 업스트림 브랜치를 가져왔습니다.

그 결과 다음 사진과 같이 다른 사람들의 Upstream 브랜치들과 나의 브랜치들이 섞여 깃 로그 그래프가 굉장히 보기 어렵게 됩니다.

어떻게 하면 나의 브랜치만 볼 수 있을까요?

### 해결 방법 
1. 해당 깃 프로젝트 디렉토리/.git/refs/remotes/지우고 싶은 업스트림 저장소 폴더로 이동합니다. 
   저는 원격 저장소의 이름이 upstream이므로, 앞으로는 지우고 싶은 원격 저장소가 upstream이라고 가정하고 얘기하겠습니다.
   (mac에서 파인더를 사용할 경우 `Shift + Command + .` 으로 숨김 파일을 볼 수 있습니다).
2. 해당 폴더에 자신의 브랜치를 포함한 모든 Upstream의 브랜치가 존재합니다. 원하는 브랜치를 제외한 모든 브랜치를 제거합니다. 로컬에서만 제거하는 것이기 때문에 업스트림에 있는 브랜치에는 아무런 영향이 없습니다.

```bash
# rm 파일 이름 : 해당 파일 삭제
# ls : 해당 디렉토리에 존재하는 파일들의 목록을 불러옴
# grep -v 남길 브랜치 : 앞의 실행 결과에서 원하는 브랜치를 제외한 결과만을 출력 
rm $(ls | grep -v 남길 브랜치 | grep -v 남길 브랜치)
```

이후 터미널에서 깃 로그를 보면 원하는 브랜치만 있습니다.

하지만 다시 Sourcetree를 실행하면 다른 사람들의 Upstream 브랜치가 생성됩니다. 

그러므로 다른 사람들의 Upstream 브랜치를 가져오지 않게 `.git/config` 파일을 수정해줘야 합니다.

```
[remote "upstream"]
  url = https://github.com/testStage/test
  fetch = +refs/heads/원하는 브랜치 이름:refs/remotes/upstream/원하는 브랜치 이름

-- 가져오고 싶은 브랜치가 1개일 경우 예시--
[remote "upstream"]
  url = https://github.com/testStage/test
  fetch = +refs/heads/ever:refs/remotes/upstream/ever
  
-- 가져오고 싶은 브랜치가 여러 개일 경우 --
[remote "upstream"]
  url = https://github.com/testStage/test
  fetch = +refs/heads/ever:refs/remotes/upstream/ever
  fetch = +refs/heads/ever2:refs/remotes/upstream/ever2
  fetch = +refs/heads/ever3:refs/remotes/upstream/ever3
```

이후 Sourcetree를 실행하면 다른 사람들의 브랜치를 가져오지 않는 것을 확인할 수 있습니다.

해당 부분과 관련하여 더 자세히 알고 싶다면 [깃 문서](https://git-scm.com/book/ko/v2/Git%EC%9D%98-%EB%82%B4%EB%B6%80-Refspec)를 참고해주세요.
