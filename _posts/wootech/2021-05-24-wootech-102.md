## MySQL 대소문자 오류
MySQL은 따로 옵션을 수정하지 않으면 테이블 생성 및 조회시 대소문자를 구분합니다.
그래서 현재 작성한 DDL 파일과 테이블 명이 일치하지 않아 오류를 발생시킨다고, 크루가 알려주었습니다.

다음의 [링크](https://bizadmin.tistory.com/entry/mysql-%ED%85%8C%EC%9D%B4%EB%B8%94-%EC%9D%B4%EB%A6%84-%EB%8C%80%EC%86%8C%EB%AC%B8%EC%9E%90-%EB%B3%80%EA%B2%BD) 에서 관련 자세한 오류를 알 수 있었고, MySQL 설정에 `lower_case_table_names = 1` 옵션을 넣어서 오류를 해결했습니다.

다음은 해결한 MySQL 설정 파일과 `docker-compose.yml` 파일입니다.

```bash
# 다음과 같이 하나의 폴더에 파일들을 생성
[SERVER][~/docker/mysql]
$ ls
custom.cnf  docker-compose.yml
```

### custom.cnf

```bash
[client]
default-character-set = utf8

[mysqld]
init_connect = SET collation_connection = utf8_general_ci
init_connect = SET NAMES utf8
character-set-server = utf8
collation-server = utf8_general_ci
lower_case_table_names = 1
sql_mode="STRICT_TRANS_TABLES,NO_ZERO_IN_DATE,NO_ZERO_DATE,ERROR_FOR_DIVISION_BY_ZERO,NO_AUTO_CREATE_USER,NO_ENGINE_SUBSTITUTION"

[mysqldump]
default-character-set = utf8

[mysql]
default-character-set = utf8
```

위의 MySQL 설정은 아직 모르는게 많네요.
알게 되면 추가하겠습니다.

### docker-compose.yml

```bash
version: '3.3'

services:
  db:
    container_name: mysql
    image: mysql:5.7
    restart: always
    ports:
     - 3306:3306
    volumes:
     - ./:/etc/mysql/conf.d
    environment:
     - MYSQL_ROOT_PASSWORD=<ROOT 패스워드>
     - MYSQL_DATABASE=<DB 이름>
     - MYSQL_USER=<USER 이름>
     - MYSQL_PASSWORD=<USER 비밀번호>
     - TZ=Asia/Seoul
```

### DataLoader
DataLoader 클래스는 서버가 실행될 때마다 역, 노선, 멤버 insert합니다. 따라서 DataLoader 클래스를 계속 존재하게 둔다면, 서버가 재실행 될때마다 UNIQUE 해야 하는 값으로 Data를 insert 하기 때문에, Duplicate 오류가 발생합니다.

제가 생각한 해결 방법은 3가지입니다.

1. DataLoader 클래스 삭제
2. Spring에서 `initialization-mode`를 이용하여, 애플리케이션를 실행할 때마다 ddl에서 drop table 하여 테이블을 모두 삭제
3. 애플리케이션을 재실행 할 때마다 DB를 재생성

저는 3번을 선택했습니다. Docker의 MySQL 컨테이너를 애플리케이션이 재실행 될때마다 Remove하고 다시 Docker 컨테이너를 생성했습니다.

```bash
# docker mysql 재시작
docker stop mysql && docker rm mysql
docker-compose up -d
```

### Nginx Config

nginx.conf 파일은 현재 다음과 같이 되어있습니다.

```bash
events {}

http {
  upstream app {
    server 172.17.0.1:8080;
  }

  access_log /var/log/nginx/access.log;
  error_log /var/log/nginx/error.log;

  # Redirect all traffic to HTTPS
  server {
    listen 80;
    return 301 https://$host$request_uri; <-- 여기
  }
	...
}
```

클라이언트가 POST로 요청 시 표시해둔 `return 301` 부분이 문제가 됩니다.

301은 리다이렉트 할 때 메소드를 유지시키지 않습니다. 예를 들어 역 생성 api인 `POST  /stations` 로 요청을 보내면, nginx가 301을 return하고, 브라우저는 응답받은 url로 GET  요청을 보냅니다.

이는 `docker logs nginx(컨테이너 이름)`으로 확인할 수 있습니다.

그래서 redirect 시 GET으로 다시 요청하는 301이 아닌 redirect하여도 메소드가 유지되는 308로 변경했습니다.

```bash
events {}

http {
  upstream app {
    server 172.17.0.1:8080;
  }

  access_log /var/log/nginx/access.log;
  error_log /var/log/nginx/error.log;

  # Redirect all traffic to HTTPS
  server {
    listen 80;
    return 308 https://$host$request_uri; <-- 여기 308로 변경
  }
	...
}
```

다음은 301, 302, 307, 308 관련 링크입니다.
- [https://perfectacle.github.io/2017/10/16/http-status-code-307-vs-308/](https://perfectacle.github.io/2017/10/16/http-status-code-307-vs-308/)

# 배포 스크립트
[조졸두님의 블로그 글](https://jojoldu.tistory.com/263) 을 참고하여, 배포 과정을 쉡 스크립트로 만들고, 배포를 자동화했습니다.

### 스크립트

```bash
#!/bin/bash

# 8080 포트를 사용중인 PID 검색
PID=$(lsof -t -i:8080)

# 만약 PID의 글자 길이가 0이면
if [ -z $PID ]
then
	echo "> 현재 구동중인 애플리케이션이 없습니다."
else
	echo "> kill -9 $PID"
	kill -9 $PID
	sleep 1
fi

echo "> REMOVE CONTAINER"

cd /home/ubuntu/docker/mysql

# docker mysql 재시작
docker stop mysql && docker rm mysql
docker-compose up -d
sleep 5

echo "> BUILD STRAT"

cd /home/ubuntu/atdd-subway-fare
./gradlew build -x test

echo "\n> DEPLOY START\n"

nohup java -jar \
-Djava.security.egd=file:/dev/./urandom \
-Dspring.profiles.active=prod \
/home/ubuntu/atdd-subway-fare/build/libs/*.jar \
1> application-log.txt 2>&1 &

echo "> DEPLOY END!"

cd ~
```
